# Руководство пользователя

## Назначение файла

Этот Excel-файл предназначен для подготовки справок по военнослужащим на основании данных из внешней выгрузки Excel и Word-шаблонов.

Книга позволяет:

- загрузить исходную выгрузку в рабочую книгу;
- найти нужного человека через форму поиска;
- подставить его данные в рабочую таблицу;
- выбрать, какие шаблоны использовать;
- сформировать готовые `.docx`-справки;
- сохранить историю выдачи на лист `IssuedDocumentsLog`;
- сохранить отдельный слепок рабочей таблицы в папку проекта.

## Основные листы

- `data`  
  Основной рабочий лист оператора. Здесь формируется список справок на выдачу.

- `ImportedData`  
  Лист с импортированной внешней выгрузкой. По нему работает поиск.

- `IssuedDocumentsLog`  
  Журнал уже сформированных справок.

- `ЧтоНового`  
  Информационный лист.

## Вкладка ленты

В книге используется пользовательская вкладка Excel:

- `Certificates`

На ней находятся все основные команды.

## Кнопки на ленте

### Блок `Certificates`

#### `Generate`

Назначение:

- запускает формирование справок по строкам листа `data`.

Что делает:

- читает строки на листе `data`;
- определяет, какие шаблоны указаны для обработки;
- открывает нужные Word-шаблоны;
- подставляет значения в плейсхолдеры;
- сохраняет готовые `.docx` в папку, выбранную через `Output Folder`;
- дописывает сведения в `IssuedDocumentsLog`;
- создает слепок рабочей таблицы в папке проекта.

Когда использовать:

- после того, как заполнены строки на листе `data`;
- после выбора папки шаблонов, списка шаблонов и папки сохранения.

Результат:

- готовые справки в выбранной выходной папке;
- запись в журнале `IssuedDocumentsLog`;
- отдельный Excel-файл со слепком данных.

#### `Search Person`

Назначение:

- открывает форму поиска человека по импортированной выгрузке.

Что делает:

- ищет совпадения по данным листа `ImportedData`;
- позволяет выбрать нужную запись;
- переносит выбранные данные в активную строку листа `data`.

Когда использовать:

- когда нужно быстро заполнить строку на листе `data` по найденному человеку.

Важно:

- перед открытием поиска желательно выделить нужную строку на листе `data`;
- если активная ячейка не выбрана, макрос подставит данные в стандартную рабочую позицию.

### Блок `Data`

#### `Import Source Data`

Назначение:

- загружает внешнюю Excel-выгрузку в лист `ImportedData`.

Что делает:

- открывает диалог выбора файла Excel;
- берет первый лист выбранной книги;
- копирует его содержимое в `ImportedData`.

Когда использовать:

- перед поиском нового человека;
- когда выгрузка обновилась и нужно работать с актуальными данными.

Важно:

- форма поиска работает именно по данным, загруженным в `ImportedData`;
- если не обновить выгрузку, поиск будет идти по старым данным.

#### `Open History`

Назначение:

- открывает лист `IssuedDocumentsLog`.

Что делает:

- переводит пользователя на журнал сформированных справок.

Когда использовать:

- чтобы посмотреть, какие справки уже были сформированы;
- чтобы проверить дату, ФИО, шаблон или другие сведения.

### Блок `Folders`

#### `Template Folder`

Назначение:

- выбирает папку, в которой лежат Word-шаблоны справок.

Что делает:

- открывает диалог выбора папки;
- сохраняет выбранный путь во внутреннюю настройку книги;
- в дальнейшем генерация берет шаблоны именно из этой папки.

Когда использовать:

- при первом запуске;
- если шаблоны были перенесены в другую папку;
- если нужно переключиться на другой комплект шаблонов.

Важно:

- в этой папке должны находиться `.docx`-шаблоны;
- список шаблонов для выбора в форме `Select Templates` читается именно из этой папки.

#### `Select Templates`

Назначение:

- открывает форму выбора шаблонов для работы.

Что делает:

- читает все `.docx` из папки, выбранной через `Template Folder`;
- показывает их списком;
- отмечает уже сохраненные шаблоны;
- после нажатия `Save` сохраняет выбранный набор во внутреннюю настройку книги.

Когда использовать:

- при первом запуске;
- когда нужно изменить набор используемых шаблонов;
- когда в папке шаблонов добавлены новые файлы.

Результат:

- выбранный набор шаблонов сохраняется во внутреннюю настройку `FILE_TEMPLATE`.

Важно:

- если папка шаблонов не выбрана, сначала нужно нажать `Template Folder`;
- в список попадают только файлы формата `.docx`.

#### `Output Folder`

Назначение:

- выбирает папку, куда будут сохраняться готовые справки.

Что делает:

- открывает диалог выбора папки;
- сохраняет путь во внутреннюю настройку книги;
- при генерации справки записываются именно туда.

Когда использовать:

- при первом запуске;
- если нужно сменить папку для сохранения готовых документов.

Важно:

- существующие файлы не затираются;
- если имя уже занято, создается новый файл с суффиксом.

### Блок `Help`

#### `About`

Назначение:

- открывает информационное окно о проекте.

Что делает:

- показывает вспомогательную форму.

Когда использовать:

- при необходимости открыть служебную информацию по книге.

## Формы

### Форма `Search Person`

Назначение:

- поиск человека по импортированной выгрузке.

Как работать:

1. Убедитесь, что данные уже загружены через `Import Source Data`.
2. Откройте форму кнопкой `Search Person` или двойным щелчком по 4-му столбцу листа `data`.
3. Введите не менее 3 символов.
4. Выберите нужную запись в списке.
5. После выбора данные будут перенесены в активную строку листа `data`.

Что переносится:

- ФИО;
- табельные/идентификационные данные;
- подразделение;
- другие связанные поля по текущей логике книги.

Важно:

- подразделение переносится с сохранением текстовых значений;
- если подразделение имеет вид `Войсковая часть 12345`, оно не теряется и корректно используется в шаблонах.

### Форма `Select Templates`

Назначение:

- выбор шаблонов, доступных для использования.

Элементы формы:

- список шаблонов;
- `Save`;
- `Cancel`.

Как работать:

1. Сначала выберите папку шаблонов через `Template Folder`.
2. Нажмите `Select Templates`.
3. Отметьте нужные шаблоны.
4. Нажмите `Save`.

Дополнительно:

- двойной щелчок по элементу списка также завершает сохранение;
- кнопка `Cancel` закрывает форму без изменений.

## Основной рабочий сценарий

Рекомендуемый порядок работы:

1. Нажать `Template Folder` и выбрать папку с Word-шаблонами.
2. Нажать `Select Templates` и выбрать нужные шаблоны.
3. Нажать `Output Folder` и выбрать папку для готовых справок.
4. Нажать `Import Source Data` и загрузить актуальную выгрузку.
5. Перейти на лист `data`.
6. Создать или выбрать рабочую строку.
7. Заполнить строку вручную или через `Search Person`.
8. Проверить значения в строке.
9. Нажать `Generate`.
10. Проверить выходную папку и лист `IssuedDocumentsLog`.

## Работа на листе `data`

На листе `data` оператор формирует строки на выдачу.

Практически это выглядит так:

- в строке указывается статус обработки;
- задается имя выходного файла;
- указываются шаблоны для обработки;
- заполняются поля человека, подразделения, вида справки и дополнительных сведений.

### Быстрое заполнение строки через поиск

Можно не открывать поиск через ribbon, а сделать так:

1. Перейти на лист `data`.
2. Дважды щелкнуть по ячейке в 4-м столбце нужной строки.
3. Откроется форма `Search Person`.

## Лист `IssuedDocumentsLog`

Это журнал созданных справок.

Что туда попадает:

- номер записи;
- дата и время создания;
- название записи;
- список шаблонов;
- данные, перенесенные из рабочей строки `data`.

Формат журнала:

- шрифт `Times New Roman`;
- размер `12`;
- тонкие границы;
- заголовок с переносом текста;
- автофильтр;
- формат даты и времени в колонке `Created On`.

## Что создается после генерации

После нажатия `Generate` обычно создаются три результата:

- готовые `.docx`-файлы в папке, выбранной через `Output Folder`;
- запись в `IssuedDocumentsLog`;
- отдельный Excel-файл-слепок рабочей таблицы в папке проекта.

## Важные правила работы

- Сначала выбирайте папку шаблонов, затем список шаблонов.
- Перед поиском человека обновляйте выгрузку через `Import Source Data`.
- Проверяйте активную строку на листе `data` перед вставкой найденной записи.
- Если шаблонов в папке нет, форма `Select Templates` покажет предупреждение.
- Если имя выходного файла уже существует, книга не затирает старый файл, а создает новый вариант.

## Типовые ошибки пользователя

### Поиск не находит человека

Причины:

- не загружена выгрузка;
- загружена старая выгрузка;
- введено слишком короткое значение для поиска.

Что делать:

- нажать `Import Source Data`;
- повторить поиск;
- вводить не менее 3 символов.

### В форме `Select Templates` нет шаблонов

Причины:

- не выбрана папка шаблонов;
- в папке нет файлов `.docx`;
- выбрана не та папка.

Что делать:

- нажать `Template Folder`;
- выбрать правильную папку;
- открыть `Select Templates` повторно.

### Справки не появляются в нужной папке

Причины:

- не выбрана выходная папка;
- была выбрана старая папка.

Что делать:

- нажать `Output Folder`;
- выбрать нужную директорию;
- повторить генерацию.

## Краткая памятка

Если нужно быстро вспомнить порядок:

1. `Template Folder`
2. `Select Templates`
3. `Output Folder`
4. `Import Source Data`
5. `Search Person` или заполнение строки вручную
6. `Generate`
7. `Open History`
